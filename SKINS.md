# Документация по формату скинов

## Структура файлов

Скины хранятся в директории `public/skins/` и организованы по типам:

```
public/skins/
├── hat/          # Шапки
├── glasses/      # Очки
├── scarf/        # Шарфы
├── shirt/        # Кофты
└── pants/        # Штаны
```

## Формат файла скина

Каждый скин представляет собой JSON файл с расширением `.json`.

### Пример структуры:

```json
{
  "name": "Красная шапка",
  "description": "Теплая красная шапка",
  "color": "#FF0000",
  "model": "hat_red.glb",
  "texture": "hat_red.png",
  "position": {
    "x": 0,
    "y": 1.2,
    "z": 0
  },
  "scale": {
    "x": 1,
    "y": 1,
    "z": 1
  },
  "rotation": {
    "x": 0,
    "y": 0,
    "z": 0
  }
}
```

### Поля:

- **name** (string, обязательное) - Название скина
- **description** (string, опциональное) - Описание скина
- **color** (string, опциональное) - HEX цвет для простых скинов (например, "#FF0000")
- **model** (string, опциональное) - Путь к 3D модели (формат .glb или .gltf)
- **texture** (string, опциональное) - Путь к текстуре (формат .png или .jpg)
- **position** (object, опциональное) - Позиция скина относительно персонажа
  - x, y, z (number) - Координаты
- **scale** (object, опциональное) - Масштаб скина
  - x, y, z (number) - Масштаб по осям
- **rotation** (object, опциональное) - Поворот скина
  - x, y, z (number) - Углы поворота в радианах

## Типы скинов

### 1. Hat (Шапка)
- Позиция: обычно y = 1.2 (на голове)
- Пример: `public/skins/hat/red_hat.json`

### 2. Glasses (Очки)
- Позиция: обычно y = 1.1, z = 0.15 (перед лицом)
- Пример: `public/skins/glasses/sunglasses.json`

### 3. Scarf (Шарф)
- Позиция: обычно y = 0.8 (на шее)
- Пример: `public/skins/scarf/blue_scarf.json`

### 4. Shirt (Кофта)
- Позиция: обычно y = 0.4 (на теле)
- Пример: `public/skins/shirt/winter_jacket.json`

### 5. Pants (Штаны)
- Позиция: обычно y = 0 (на ногах)
- Пример: `public/skins/pants/jeans.json`

## Создание нового скина

### Шаг 1: Создайте JSON файл

Создайте файл в соответствующей директории типа скина. Например:
```
public/skins/hat/my_custom_hat.json
```

### Шаг 2: Заполните данные

```json
{
  "name": "Моя кастомная шапка",
  "description": "Уникальная шапка",
  "color": "#00FF00",
  "position": {
    "x": 0,
    "y": 1.2,
    "z": 0
  }
}
```

### Шаг 3: Добавьте 3D модель (опционально)

Если у вас есть 3D модель:
1. Поместите файл модели в ту же директорию
2. Укажите путь в поле `model`:
```json
{
  "model": "my_custom_hat.glb"
}
```

### Шаг 4: Добавьте текстуру (опционально)

Если у вас есть текстура:
1. Поместите файл текстуры в ту же директорию
2. Укажите путь в поле `texture`:
```json
{
  "texture": "my_custom_hat_texture.png"
}
```

## Простые скины (только цвет)

Для быстрого создания скина можно использовать только поле `color`:

```json
{
  "name": "Синяя шапка",
  "color": "#0000FF"
}
```

Такой скин будет применен как простая цветная модификация соответствующей части персонажа.

## 3D модели

### Поддерживаемые форматы:
- `.glb` (рекомендуется) - бинарный формат GLTF
- `.gltf` - текстовый формат GLTF

### Требования к моделям:
- Размер файла: рекомендуется до 1MB
- Полигоны: рекомендуется до 5000 для мобильных устройств
- Текстуры: формат PNG или JPG, размер до 1024x1024

### Инструменты для создания:
- Blender (бесплатно)
- Maya, 3ds Max (платно)
- Онлайн редакторы: Tinkercad, Sketchfab

## Примеры скинов

См. примеры в директории `public/skins/` для каждого типа.

## Применение скинов

Скины применяются автоматически при загрузке персонажа. Игрок может менять скины через интерфейс игры.

## Обновление без перезапуска

Новые скины, добавленные в директорию `public/skins/`, автоматически становятся доступными без перезапуска сервера. Просто обновите страницу в приложении.
